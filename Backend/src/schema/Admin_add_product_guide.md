# Admin — Add Product Guide

## What the admin fills in

### Step 1 — Basic Info
| Field | Input Type | Required | Notes |
|---|---|---|---|
| Product name | Text input | Yes | e.g. "Nakshi Kantha Panjabi" |
| Category | Nested dropdown | Yes | Fetched from API. Shows full path e.g. Clothing > Menswear > Panjabi |
| Price (৳) | Number input | Yes | Single price for all sizes |
| SKU | — | Auto | Generated by backend from product_id. Admin never fills this in. |
| Description | Textarea | No | Product story, materials, care instructions |

---

### Step 2 — Sizes & Stock
Two scenarios depending on product type.

**Unsized product (Saree, Wall Art, Pottery)**
Admin sees a single stock field — no size selector needed. The default NULL variant was auto-created when the product was inserted.

| Field | Input Type | Notes |
|---|---|---|
| Stock quantity | Number input | Updates the auto-created NULL variant |

**Sized product (Panjabi, Kurti, Shirt)**
Admin adds sizes one by one. Each size gets its own stock count.

| Field | Input Type | Notes |
|---|---|---|
| Has sizes? | Toggle/Checkbox | Switches between unsized and sized UI |
| Size label | Text input | S, M, L, XL, Free Size, etc. |
| Stock quantity | Number input | Per size |
| + Add another size | Button | Adds a new size row |
| Remove | Button (per row) | Removes that size row |

> At least one size row must exist before the product can be saved.

---

### Step 3 — Images
| Field | Input Type | Notes |
|---|---|---|
| Upload images | File picker (multi) | Accepts JPG, PNG, WebP. Max 5MB each |
| Primary image | Radio button per image | One must be selected as primary |
| Remove | Button per image | Removes that image |

> The primary image is shown in product listings. All images are shown on the product detail page.

---

## UI Flow — Step by step

```
[ Basic Info ] → [ Sizes & Stock ] → [ Images ] → [ Review & Save ]
```

Each step is a separate section or tab. Admin can go back and edit any step before saving.

---

## Buttons

| Button | Where | Action |
|---|---|---|
| Next | Bottom of each step | Validates current step and moves forward |
| Back | Bottom of each step | Goes to previous step without losing data |
| + Add Size | Step 2 (sized products) | Adds a new size input row |
| Remove | Next to each size row | Removes that size |
| Upload Images | Step 3 | Opens file picker |
| Set as Primary | Below each uploaded image | Marks that image as primary |
| Remove | Below each uploaded image | Deletes that image |
| Save Product | Final step | Submits everything |
| Save as Draft | Final step | Saves with `is_active = false` |
| Cancel | Top right always | Discards everything and goes back to product list |

---

## Validation — what to check before saving

### Step 1
- Product name is not empty
- Category is selected
- Price is a positive number

### Step 2
- At least one variant exists (either the NULL variant with stock, or at least one sized variant)
- No duplicate size labels
- Stock quantity is 0 or more (not negative)

### Step 3
- At least one image uploaded
- Exactly one image marked as primary

---

## What happens in the backend on save

1. `POST /products` — inserts the product row without SKU. Trigger auto-creates a NULL variant.

2. Auto-generate SKU from the returned `product_id` and update immediately:
```js
const productId = result.rows[0].product_id;
const sku = 'FM-' + String(productId).padStart(6, '0');
// FM-000001, FM-000042, FM-000100 etc.

await pool.query(
  'UPDATE product SET sku = $1 WHERE product_id = $2',
  [sku, productId]
);
```
SKU is guaranteed unique because it is derived from `product_id` which is a SERIAL — no two products can ever share the same `product_id`.

3. If sized:
   - `DELETE` the NULL variant that was auto-created by the trigger
   - `POST /products/:id/variants` for each size the admin added
4. If unsized:
   - `PATCH /products/:id/variants/:variantId` — update stock on the NULL variant
5. `POST /products/:id/images` — upload each image to Cloudinary, store returned URL
6. Set `is_primary = true` on the chosen image

If any step fails, the product should be rolled back or flagged as incomplete.

## UX Notes

- Category dropdown should show the full path, not just the leaf name. "Panjabi" is ambiguous — "Clothing > Menswear > Panjabi" is not.
- Price input should show the ৳ symbol prefix.
- Stock input should not allow decimals.
- SKU is never shown on the add product form — it is generated automatically and can be displayed after save on the product detail page.
- After uploading images show a thumbnail preview immediately — do not wait for save.
- If admin tries to leave the page with unsaved changes, show a confirmation dialog.
- After successful save, redirect to the product detail page so admin can verify everything looks correct.
- "Save as Draft" is important — admin may want to fill in info across multiple sessions without making the product live.
